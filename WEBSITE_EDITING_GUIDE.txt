4. CREATING NEW HTML PAGES
================================================================================

FOR A NEW BOOK PAGE:
--------------------
1. Copy the template file:
   books/the-day-when-servants-reign.html

2. Rename it to match your book slug:
   books/your-book-slug.html

3. Edit ONLY these lines in the <head> section:

   <title>Your Book Title - Book X of Series Name</title>
   <meta name="description" content="Your book description here">

4. That's it! JavaScript loads everything else from data.js


FOR A NEW SERIES PAGE:
----------------------
1. Copy the template file:
   series/severance-chronicles.html

2. Rename it to match your series slug:
   series/your-series-slug.html

3. Edit these sections:

   A. In <head>:
      <title>Your Series Name - Genre by Sid Lancaster</title>
      <meta name="description" content="Your series description">

   B. In the hero section header:
      <div class="section-header">
          <h2>Your Series Name</h2>
      </div>

   C. In the breadcrumb:
      <li class="breadcrumb-item active">Your Series Name</li>

4. Done! JavaScript handles the rest

7. IMPORTANT NOTES & BEST PRACTICES
================================================================================

SLUG NAMING RULES:
------------------
‚úì Use kebab-case (lowercase with hyphens)
‚úì No spaces, no underscores, no capital letters
‚úì The slug MUST match in three places:
    1. The key in booksData/seriesData
    2. The slug field inside the object
    3. The HTML filename

CORRECT:
    'the-day-when-servants-reign': {
        slug: 'the-day-when-servants-reign',
    }
    File: books/the-day-when-servants-reign.html

INCORRECT:
    'The Day When Servants Reign'        ‚ùå Capital letters
    'the_day_when_servants_reign'        ‚ùå Underscores
    'the day when servants reign'        ‚ùå Spaces


DATA TYPE RULES:
----------------
Arrays (use square brackets):
    themes: ["Theme 1", "Theme 2", "Theme 3"]           ‚úì Correct
    themes: "Theme 1, Theme 2, Theme 3"                 ‚úó Wrong

Numbers (no quotes):
    wordCount: 56000                                    ‚úì Correct
    wordCount: "56000"                                  ‚úó Wrong

Booleans (no quotes):
    published: false                                    ‚úì Correct
    published: "false"                                  ‚úó Wrong

Strings (use quotes):
    title: "The Day When Servants Reign"               ‚úì Correct
    title: The Day When Servants Reign                 ‚úó Wrong


CHARACTER REQUIREMENTS:
-----------------------
‚Ä¢ Always provide exactly 4 traits per character
‚Ä¢ Each character needs: name, role, description, traits array
‚Ä¢ Minimum 1 character, recommended 2-6 main characters


SAMPLE CHAPTER FORMAT:
----------------------
‚Ä¢ Each paragraph is a separate string in the array
‚Ä¢ Use double quotes for strings
‚Ä¢ Escape single quotes inside text with backslash: \'
  Example: "He didn\'t know what to do"


PROGRESS MILESTONES:
--------------------
Use these exact stage names (case-sensitive):
    ‚Ä¢ Prewriting
    ‚Ä¢ Drafting
    ‚Ä¢ Revising
    ‚Ä¢ Editing
    ‚Ä¢ Publishing
    ‚Ä¢ Published

Mark completed stages with:
    completed: true,
    date: '2024-08',


KEEPING THINGS ORGANIZED:
--------------------------
‚ñ° Update data.js regularly with writing progress
‚ñ° Keep content warnings accurate and up-to-date
‚ñ° Use consistent formatting (tabs, spacing)
‚ñ° Test changes by viewing pages in a browser
‚ñ° Back up data.js before making major changes
‚ñ° Update lastUpdated field when making changes


TESTING YOUR CHANGES:
----------------------
1. Save data.js after editing
2. Open the HTML page in a web browser
3. Check that all data displays correctly
4. Test on mobile view (resize browser window)
5. Check navigation dropdowns work


COMMON MISTAKES TO AVOID:
--------------------------
‚úó Forgetting to add comma after object properties
‚úó Using single quotes inside double-quoted strings without escape
‚úó Misspelling stage names in progressMilestones
‚úó Forgetting to update navigation after adding books/series
‚úó Using wrong path separator (use / not \)
‚úó Not providing exactly 4 traits for characters


================================================================================
TROUBLESHOOTING
================================================================================

BOOK/SERIES NOT SHOWING UP:
----------------------------
‚Üí Check that slug matches in all three places (key, slug field, filename)
‚Üí Verify data.js doesn't have syntax errors (missing commas, quotes)
‚Üí Make sure the HTML file exists in correct folder
‚Üí Check browser console for JavaScript errors (F12)


NAVIGATION LINKS BROKEN:
-------------------------
‚Üí Verify file paths are correct (use ../ to go up one directory)
‚Üí Check filename matches exactly (case-sensitive)
‚Üí Ensure .html extension is included


DATA NOT UPDATING:
------------------
‚Üí Hard refresh browser: Ctrl+F5 (Windows) or Cmd+Shift+R (Mac)
‚Üí Check that you saved data.js after editing
‚Üí Verify no JavaScript syntax errors in data.js


FORMATTING LOOKS WRONG:
-----------------------
‚Üí Check that you didn't accidentally edit styles.css
‚Üí Verify HTML structure matches template files
‚Üí Test in different browser to rule out caching issues
